❓ 常见问题 FAQ
AI Agent 和 IDE 插件（Copilot）有什么区别？▼
IDE 插件是"被动建议"——你写代码它补全；Agent是"主动执行"——你描述需求，它自动读取文件、分析代码、生成修改、写入文件。Agent 能完成多步骤任务，插件只能做单步建议。类比：插件是副驾驶，Agent 是自动驾驶。
开源 Agent 和闭源 Agent 怎么选？▼
开源优势：免费、可自托管、数据不出本机、支持多模型切换。闭源优势：开箱即用、无需运维、体验更流畅。如果你是个人开发者且在意成本，选开源+国内 API；如果是团队且追求效率，选闭源（Cursor/Copilot）。
Agent 会不会自动删除/覆盖我的代码？▼
有可能，但可以防范。大多数 Agent 默认开启"自动应用修改"，这意味着它改的代码会直接写入文件。建议：①始终使用 Git，出问题可回滚；②生产环境关闭自动应用，改为手动确认；③限制 Agent 的文件访问范围（沙箱模式）。
Devin 值得 $500/月吗？▼
对大多数人不值得。Devin 定位是全自主 AI 软件工程师，$500/月约为 Cursor Pro $20 的 25 倍。Codex（ChatGPT Pro $200/月）是另一档产品。除非你的时薪 > $50 且经常需要处理大量重复性编程任务，否则 Hermes/Cline + GLM-5.2 或 DeepSeek-V4 API 的组合更划算。
MCP 是什么？为什么重要？▼
MCP（Model Context Protocol）是 Anthropic 提出的统一工具调用协议，让 Agent 能连接数据库、浏览器、文件系统等外部工具。重要性：MCP 正在成为行业标准，选支持 MCP 的 Agent 意味着未来可以接入更多工具，不会被生态锁定。
哪个 Agent 写代码最强？▼
取决于底层模型。用 Claude Opus 时，Claude Code 最强；用 GPT-5 时，Copilot Workspace 最强。目前 SWE-bench 得分排名：Claude Code > Cursor Agent > OpenClaw > Cline。但差距在缩小，开源 Agent 配合强模型已能达到闭源 80-90% 的效果。

📈 Agent 趋势分析

从"对话助手"到"自主执行"，AI Agent 正在重塑编程方式。

1️⃣ 从"辅助"到"自主" — Agent 演进三阶段
• 阶段一（2023）：代码补全 — Copilot 模式，一行一行建议
• 阶段二（2024-2025）：对话式编程 — Chat 模式，多文件编辑
• 阶段三（2026）：自主执行 — Agent 模式，理解需求→拆解任务→执行→验证→提交
→ 现在正处于阶段三的爆发期，Hermes / Claude Code / Cursor Agent 代表了这个方向
2️⃣ MCP 协议 — 统一 Agent 工具调用
• 什么是 MCP：Model Context Protocol，Anthropic 推出的开放协议，让 Agent 统一调用外部工具
• 为什么重要：之前每个 Agent 自己造轮子连接数据库/API/文件系统，MCP 统一后工具生态共享
• 现状：Hermes/OpenClaw/Cline/Cursor/Claude Code 已支持，IDE Agent 纷纷接入
→ 选 Agent 优先选支持 MCP 的，生态红利才刚开始
3️⃣ 开源 Agent 生态爆发
• 趋势：Hermes 已成 2026 开源 Agent 实战首选（记忆/技能/IM 完整度领先）；OpenClaw Stars 仍高但新增用户更多选 Hermes
• 模型侧：GLM-5.2、DeepSeek-V4-Pro 全量开放后，开源 Agent + 国产 Coding Plan 月成本 < ¥50
• 风险：开源项目维护靠社区，稳定性不如商业产品，生产环境需谨慎
4️⃣ 安全与权限 — Agent 的达摩克利斯之剑
• 核心矛盾：Agent 越自主，权限越大，风险越高 — 可以帮你写代码，也可以帮你删库
• 安全实践：沙箱执行 → 人工确认 → 分级授权 → 审计日志
• 行业趋势：企业级 Agent（Copilot Workspace/ArkClaw）内置权限管控，开源 Agent 需自行配置
→ 永远不要给 Agent 无限权限，即使它说"我能搞定"
