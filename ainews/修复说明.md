# 数据库写入问题修复说明

## 问题描述

其他监控脚本（如 `ai-bot-news.py`, `openai-news.py`, `anthropic-news.py` 等）的内容在更新，但没有写入数据库。

## 根本原因

这些脚本的逻辑是：
1. **内容比对一致时**：直接 `return` 或 `continue`，**不会尝试写入数据库**
2. **内容有更新时**：才会尝试写入数据库
3. **如果写入失败**：异常被捕获但可能没有详细输出

这导致：
- 如果之前写入失败，即使内容更新了也不会再次尝试
- 如果内容一致，即使数据库中没有记录也不会写入

## 已修复的脚本

以下脚本已更新，现在**即使内容比对一致也会尝试入库**：

1. ✅ `ai-bot-news.py`

## 修复内容

### 1. 增强"内容一致"时的处理

**修复前**：
```python
if old_content == news_text:
    print('内容比对一致，无需更新和推送')
    return  # 直接返回，不写入数据库
```

**修复后**：
```python
if old_content == news_text:
    print('内容比对一致，无需更新和推送')
    # 即使内容一致，也检查是否需要入库（可能之前入库失败）
    print("检查是否需要入库...")
    try:
        # 尝试入库（如果已存在会被去重）
        success = push_logger.log_push(...)
        if success:
            print(f"✓ 已入库（之前可能未入库）...")
        else:
            print(f"✓ 内容已存在于数据库，无需重复入库")
    except Exception as e:
        print(f"⚠ 检查入库时出错: {str(e)}")
    return
```

### 2. 增强错误处理

**修复前**：
```python
except Exception as e:
    print(f"记录入库失败: {str(e)}")
```

**修复后**：
```python
except Exception as e:
    print(f"记录入库失败: {str(e)}")
    import traceback
    print(f"详细错误信息:")
    traceback.print_exc()
```

## 工作原理

1. **内容一致时**：
   - 仍然会尝试调用 `push_logger.log_push()`
   - 如果数据库中没有记录，会成功写入
   - 如果数据库中已有记录（去重），会返回 `False`，但这是正常的

2. **内容更新时**：
   - 正常写入数据库
   - 如果失败，会输出详细的错误信息

3. **去重逻辑**：
   - `push_logger.log_push()` 内部会检查：
     - 内容哈希是否重复
     - URL是否重复
     - 标题+来源组合是否相似
   - 如果重复，返回 `False`，但不会报错
