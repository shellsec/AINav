# AI资讯日报系统使用说明

## 系统概述

AI资讯日报系统是在原有监控推送系统基础上新增的功能，用于记录每天的推送内容并生成日报、周报和数据分析报告。

## 新增功能

### 1. 推送日志记录 (`push_logger.py`)
- 自动记录每次推送的详细信息
- 存储到SQLite数据库中
- 支持去重检查，避免重复记录
- 记录内容包括：来源、标题、内容、分类、链接、推送时间、推送渠道等

### 2. 日报生成器 (`daily_report_generator.py`)
- 生成每日AI资讯汇总报告
- 支持Markdown、纯文本、JSON三种格式
- 按来源分组展示资讯
- 包含统计信息和内容摘要

### 3. 数据分析工具 (`analytics.py`)
- 提供多维度数据分析
- 活跃度统计、推送成功率、时间分布等
- 支持导出CSV数据
- 生成可视化分析报告

### 4. 自动报告生成器 (`auto_report_generator.py`)
- 支持定时自动生成报告
- 可配置生成时间和频率
- 支持手动触发生成
- 可选择推送报告摘要

## 安装和配置

### 1. 安装依赖
```bash
pip install schedule pandas matplotlib
```

### 2. 配置文件
复制并编辑配置文件：
```bash
copy config.ini.example config.ini
```

在 `config.ini` 中添加报告相关配置：
```ini
[report]
# 是否自动推送报告摘要
auto_push = false
# 日报生成时间
daily_time = 18:00
# 周报生成日期
weekly_day = friday
# 周报生成时间
weekly_time = 17:00
```

### 3. 更新现有脚本
运行批量更新脚本，为现有监控脚本添加日志记录功能：
```bash
python update_scripts_with_logging.py
```

## 使用方法

### 手动生成报告

#### 生成今日日报
```bash
python auto_report_generator.py daily
```

#### 生成本周周报
```bash
python auto_report_generator.py weekly
```

#### 生成数据分析报告
```bash
python auto_report_generator.py analytics
```

#### 生成所有报告
```bash
python auto_report_generator.py all
```

### 启动定时任务
```bash
python auto_report_generator.py schedule
```

### 直接使用各个模块

#### 使用日报生成器
```python
from daily_report_generator import DailyReportGenerator
from datetime import date

generator = DailyReportGenerator()

# 生成今日日报
daily_report = generator.generate_daily_report()
print(daily_report)

# 生成本周周报
weekly_report = generator.generate_weekly_report()
print(weekly_report)
```

#### 使用数据分析工具
```python
from analytics import PushAnalytics

analytics = PushAnalytics()

# 获取活跃度统计
stats = analytics.get_activity_stats(30)
print(stats)

# 生成分析报告
report = analytics.generate_analytics_report(30)
print(report)
```

```

## 报告内容说明

### 日报内容
- 当日推送的所有AI资讯
- 按来源分组展示
- 包含标题、分类、发布时间、链接等信息
- 统计各来源的资讯数量

### 周报内容
- 最近7天的资讯汇总
- 来源活跃度统计
- 按日期展示每日资讯
- 包含百分比统计

### 分析报告内容
- 来源活跃度统计
- 推送成功率分析
- 内容分类统计
- 推送时间分布
- 每日推送趋势

## 数据库结构

推送记录表 (`push_records`)：
- `id`: 主键
- `source`: 来源（OpenAI、Anthropic等）
- `title`: 标题
- `content`: 完整内容
- `content_hash`: 内容哈希（用于去重）
- `push_time`: 推送时间
- `push_date`: 推送日期
- `category`: 分类
- `url`: 链接
- `push_channel`: 推送渠道
- `push_status`: 推送状态
- `keywords`: 关键词（JSON格式）

## 注意事项

1. **数据库文件**: 系统会自动创建 `push_logs.db` 文件，请勿删除
2. **报告目录**: 报告会保存在 `reports/` 目录下，请确保有写入权限
3. **备份文件**: 更新脚本时会创建 `.backup` 备份文件，测试无误后可删除
4. **依赖库**: 确保安装了所有必需的Python库
5. **编码问题**: 所有文件都使用UTF-8编码，确保系统支持中文

## 故障排除

### 常见问题

1. **数据库锁定错误**
   - 确保没有多个程序同时访问数据库
   - 重启相关脚本

2. **报告生成失败**
   - 检查 `reports/` 目录是否存在且有写入权限
   - 确保数据库中有数据

3. **推送日志记录失败**
   - 检查 `push_logger.py` 是否正确导入
   - 确保数据库文件可写

4. **定时任务不执行**
   - 检查配置文件中的时间格式
   - 确保系统时间正确

2. 测试报告生成：

```

## 扩展功能

系统设计为模块化，可以轻松扩展：

1. **新增推送渠道**: 在 `push_logger.py` 中添加新的推送渠道记录
2. **自定义报告格式**: 在 `daily_report_generator.py` 中添加新的格式支持
3. **更多分析维度**: 在 `analytics.py` 中添加新的分析功能
4. **推送报告**: 在 `auto_report_generator.py` 中完善推送功能
