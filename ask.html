AINavAI第一思考Ask

💬 Ask — 问对问题，比得到答案更重要

AI时代最重要的能力。不是AI不行，是你没问对。
✨ Ideate 💬 Ask 🗺️ Plan 🔧 Debug 🤖 Agent
🔥 痛点 — 为什么你用AI总是不满意
痛点1：模糊提问 → 废话输出
症状：你输入"帮我写个网站"，AI给你一堆泛泛的模板和建议。

真相：不是AI能力差，是你没说清楚要什么。就像你对装修师傅说"帮我装修"，他能做什么？

核心问题：模糊的想法产生模糊的输出。AI不会猜你的心思——它只会精确执行你说的每一句话。
痛点2：一次问太多 → AI也乱
症状："帮我从零做一个电商App，要有用户注册、商品列表、购物车、支付、后台管理……" → AI东一块西一块，什么都做了什么都不好。

真相：AI的注意力是有限的。一次塞10个需求，等于没有重点。

核心问题：人脑一次也只能处理3-5个概念，AI也一样。拆分是效率的前提。
痛点3：不给上下文 → AI瞎猜
症状："帮我写个文案" → AI写出来的跟你的产品完全无关。

真相：AI不知道你是谁、卖什么、给谁看。它只能从"文案"二字出发，写一个最"平均"的结果。

核心问题：没有上下文的提问，等于让AI在无限可能中随机选一个。概率上，它选的不会是你想要的。
落地技巧：把赛制、编码约定、禁止事项写进 CLAUDE.md / Cursor Rules / /rules，少每次重复解释。Capybara 把 VibeJam 规则直接丢进规则目录——见 总览 · 实战案例。
痛点4：不会追问 → 换AI重来
症状：AI第一次回答不满意，你就换个AI重新问，结果更差。

真相：追问比换工具有效10倍。AI第一次回答已经理解了你的一部分意图，追问是在这个基础上优化。换AI是从零开始。

核心问题：把AI当搜索引擎用 → 一次性。把AI当协作者用 → 迭代式。后者效率高5-10倍。
📋 案例 — 同一个需求，不同的问法
📄 案例1：做产品宣传视频
❌ 笼统问（Lv.1）
"帮我做个产品宣传视频"

AI输出："好的，以下是制作产品宣传视频的步骤：1.确定目标受众 2.编写脚本 3.拍摄素材 4.后期剪辑……"
→ 废话，毫无可用输出
✅ 精准问（Lv.5）
"你是一个短视频导演。我的产品是AI编程助手'CodeBuddy'，目标用户是1-3年经验的前端开发者，核心卖点是'AI自动补全+Agent模式自动写代码'。请写一个30秒TikTok风格的分镜脚本，要求：前3秒用'还在手动写代码？'抓眼球，中间15秒展示AI自动补全+Agent执行，结尾CTA'免费试用'。输出格式：秒数+画面+旁白。"
→ 直接可用的分镜脚本
提效倍率：5x 可用性：从0% → 80% 修改轮次：从5轮 → 1轮
💻 案例2：写代码功能
❌ 笼统问（Lv.1）
"帮我写个登录功能"

AI输出：给你一个最基础的HTML表单+PHP后端，跟你的项目完全无关。
→ 代码不可用，重新来
✅ 精准问（Lv.5）
"我的项目是Next.js 14 + Prisma + NextAuth。请在 app/api/auth/ 下添加 Google OAuth 登录，使用 Prisma Adapter，用户表已有 email/name/image 字段。要求：1.只改必要的文件 2.加上环境变量示例 3.不用写测试。"
→ 直接复制粘贴可用
提效倍率：10x 一次通过率：从20% → 85%
🎨 案例3：AI出图
❌ 笼统问（Lv.1）
"画一个科技感的封面"

AI输出：一张蓝色渐变+几个圆形光斑的通用图，跟"科技感"最俗的刻板印象一样。
→ 看着像模板
✅ 精准问（Lv.5）
"16:9横版封面，深色背景(#0a0a0a)，左侧放产品截图(带轻微3D旋转)，右侧留出标题区域，底部加一行小字副标题。风格参考 Linear.app 官网的极简设计——线条、渐变光带、玻璃质感。不要出现人物。"
→ 一次出图接近终稿
提效倍率：8x 重出次数：从5次 → 1-2次
🛠️ 落地 — Ask的5个层级 + 万能模板
Ask的5个层级
层级你说的AI给的可用性适用场景
Lv.1"帮我做个视频"泛泛建议/步骤❌ 0-10%探索阶段，还不知道要什么
Lv.2"做个30秒产品视频，科技感"有框架的输出⚠️ 20-30%有方向但缺细节
Lv.3"参考这个视频的风格，做类似的"近似结果✅ 40-60%有参考物
Lv.4"先分镜→再文案→再画面→再配音"可控流程+每步输出✅ 60-80%复杂任务拆步骤
Lv.5角色+任务+约束+示例+格式精准可用结果🎯 80-95%任何需要AI输出的场景

每升一级，AI输出的可用性提升3-5倍。从Lv.1到Lv.5的差距不是5倍，是50倍。
万能Ask公式（可复制）
你是一个[角色]。 请帮我[任务]。 要求：[约束1]，[约束2]，[约束3]。 参考风格：[示例/链接/截图描述]。 输出格式：[JSON / Markdown / 代码 / 分镜脚本 / 表格]。 不要：[明确不做的事]。 不确定时先问我最多 3 个澄清问题，再产出。

公式：角色 + 任务 + 约束 + 示例 + 格式。下面是填好的场景版，可直接改括号内容。

编程场景模板
你是一个高级全栈工程师。
请在 app/api/users/ 下新增一个 GET 接口，返回分页用户列表。
技术栈：Next.js 14 + Prisma + PostgreSQL。
要求：1.支持 page/size 参数 2.返回 total+data 3.加 Zod 校验。
参考：现有 app/api/auth/ 的写法风格。
输出：只输出新增的 route.ts 文件，不用解释。
文案场景模板
你是一个科技产品文案专家。
请帮我写 3条小红书种草文案。
产品：AI编程助手CodeBuddy，核心卖点：Agent模式自动写代码。
要求：1.每条200字内 2.开头用痛点抓眼球 3.结尾加CTA 4.语气轻松不说教。
参考：Apple产品文案的简洁风格。
输出：3条文案，每条单独标号。
视频场景模板
你是一个短视频导演。
请帮我写 30秒TikTok风格的分镜脚本。
产品：XX，目标用户XX，核心卖点XX。
要求：1.前3秒用XX抓眼球 2.中间展示核心功能 3.结尾CTA。
参考：XX品牌TikTok账号的风格。
输出：秒数 + 画面描述 + 旁白文案 + BGM建议。
分析场景模板
你是一个行业分析师。
请帮我分析 2026年AI编程工具市场。
关注：1.市场规模与增速 2.头部玩家及份额 3.技术趋势 4.投资机会。
要求：1.数据标注来源 2.有争议的标明 3.结论要可操作。
输出：Markdown，含数据表格+结论摘要。
🔄 闭环 — Ask不是一次性的，是迭代的
Ask的闭环流程
① 初次Ask → ② 审阅输出 → ③ 标注问题 → ④ 追问修正 → ①' 迭代Ask

关键原则：3轮对话内得到满意结果 = 合格。
超过3轮还没满意 → 不是Ask的问题，是你在Lv.1级问Lv.5级的问题。回去补充上下文。
追问的3种方式
  • 补充式："这个方向对，但请加上[缺少的部分]"。→ 最常用，解决60%的情况
  • 修正式："第2段语气太正式了，请改成[轻松/专业/口语]风格"。→ 定向修改，解决30%
  • 转向式："这个方案不合适，换一种思路：[新方向]"。→ 3次修不好就转向，解决10%
追问 ≠ 从头问。每次追问都带上前文的上下文。
闭环的验收标准
场景合格标准不合格信号
代码复制粘贴→跑起来→功能正确还需要改3处以上才能跑
文案只需微调措辞即可发布需要重写超过50%
分镜可以直接按脚本拍摄/制作缺少关键画面或时长不对
出图一次出图接近终稿需要重出3次以上
分析结论可直接用于决策数据存疑或结论太泛

如果你3轮Ask都达不到合格标准 → 回到模板，从Lv.5重新问。
⚠️ 3次修不好就换方案
死磕是最大的浪费。

如果你追问了3次AI还是给不出满意结果，问题可能不在AI的能力，而是：
  • 你的需求本身就矛盾（"要简单又要功能全"）
  • 你问的方案不是最优解（换个思路可能3秒搞定）
  • 这个任务AI确实做不了（该人做的就人做）
Ask的终极智慧：知道什么该问AI，什么不该问。
