💡 目录小贴士
  • 全局目录 — 所有项目共享的通用 Skill（如 Karpathy 极简思维、Addy 工程规范）
  • 项目目录 — 项目特定的 Skill（如某个 React 项目的组件规范）
  • 优先级 — 项目级 > 全局级；同类 Skill 只保留一个，避免冲突

📝 通用 Skill 模板
通用 SKILL.md 模板 — 直接复制修改
# 项目名称 Skill

## 角色定义
你是一个专业的 [技术栈] 开发者，擅长 [核心能力]。

## 编码规范
- 使用 [语言] 的最新稳定特性
- 遵循 [风格指南] 命名规范
- 优先使用 [库/框架] 解决问题

## 工作流程
1. 需求分析：理解用户意图，确认边界条件
2. 方案设计：先描述方案，确认后再编码
3. 编码实现：按模块逐步实现，每步验证
4. 测试验证：编写对应测试，确保覆盖

## 约束条件
- 不使用已弃用的 API
- 必须处理错误和边界情况
- 代码必须有类型注解（如适用）

## 输出格式
- 修改时只输出变更部分
- 附带简要修改说明
- 标注可能的风险点
🔑 编写 Skill 的 5 个关键技巧
  • 先定角色 — 告诉 AI "你是谁"，比"你要做什么"更有效
  • 约束比指令更重要 — 明确"不能做什么"比"能做什么"更能控制质量
  • 用示例代替描述 — 给 1 个好代码示例 > 写 10 行规范说明
  • 分步优于一次性 — 复杂任务拆成步骤，AI 每步确认再继续
  • 保持精简 — Skill 不是文档，300-500 行以内效果最好

❓ 常见问题
Skill 和 Prompt 有什么区别？▼
Skill 是结构化的持久化 Prompt。Prompt 是一次性的对话指令，Skill 是保存在文件中、AI 每次启动都会自动加载的行为规范。可以理解为"Prompt 的高级版"——它有角色定义、工作流程、约束条件，而且跨会话生效。
npx skills add 和 git clone 有什么区别？▼
npx skills add 是推荐的统一安装方式，它会自动识别你用的工具（Cursor/Claude Code/Codex 等），把 Skill 放到正确的目录，还支持交互式选择要装哪些。git clone 是手动方式，需要自己管理文件位置。建议优先用 npx skills add。
我该从哪个工具开始用 Skill？▼
推荐从 Claude Code 或 Cursor 开始。这两个工具的 Skill 生态最成熟、社区资源最多。Claude Code 的 SKILL.md 格式最简洁，Cursor 的 .cursorrules 用户量最大。选择你日常用的那个即可。
多个 Skill 会冲突吗？▼
可能会。如果两个 Skill 的指令矛盾（如一个要求用 TypeScript，一个要求用 Python），AI 会困惑。建议：通用规范放全局级，项目特定放项目级；同类 Skill 只保留一个。三大必装合集（Addy/Matt/Karpathy）互补不冲突，可以放心全装。
国内 GitHub 克隆慢怎么办？▼
用 gh-proxy 加速。在 GitHub 仓库 URL 前加 https://gh-proxy.com/ 前缀即可。例如：npx skills add https://gh-proxy.com/github.com/addyosmani/agent-skills --global。也可以用 LinSkills 等国内市场。
怎么更新已安装的 Skill？▼
三步搞定：① cd ~/.agents/skills（Windows: C:\Users\用户名\.agents\skills）② git pull ③ Cursor 用户执行 Reload Skills，Claude Code 用户重启 claude 即可。前提是安装时选了 Symlink 而非 Copy。
