Ideate是方向，Ask是输入，Plan是蓝图，Debug是保障，Agent是放大器
五者循环：你 Ideate → 你 Ask → AI Plan → AI Execute → 你 Debug → Agent 自动循环
方向错了，后面提效倍率越高越浪费；方向对了，每多掌握一项，提效再翻一番
熟悉 PDCA？对照一下：Ideate≈定课题 · Plan≈P · Execute/Agent≈D · Debug/验收≈C · 闭环迭代≈A — Ask 是 AI 时代把课题翻译成可执行提问的桥梁。
链路还要补上的 4 条实战原则
1. 人负责判断，AI 负责执行 — 「100% AI 写代码」说的是 Execute；选题、好玩与否、砍范围、何时停，必须人拍板（Ideate / Debug）。
2. 好玩 > 物理真实 — Ideate/Debug 时可以故意调「不真实」的参数（如食物堆更容易倒），体验优先于仿真。
3. 做不出产物，就先造工具 — Agent 放大器：AI 画不了整张地图 → 让它做地图编辑器 → 你用工具搭。这是 Vibe Coding 最高杠杆之一。
4. 多会话 + 上下文策略 — 新功能用干净会话；修 Bug 用长上下文会话；2～3 路并行各管模块，减少互相踩文件。这是 Ask/Agent 的工作流，不是玄学。
📚 实战案例 · 五步怎么走的

公开案例用来对照链路，不是背书。读完问自己：哪一步是我在做，哪一步我甩给了 AI。

🐹 Capybara 外卖游戏 · Vibe Jam 2026 冠军 完整五步

9 年 iOS 开发者，15 天、约 <$150、Claude Code 写出 2.7 万行（自称零手写），奖金 $25k。引擎 Three.js；贴图/3D/音乐另用图像与音频模型。

Ideate：一句「水豚骑摩托送外卖」→ 与 GPT/Grok 脑暴；食物改为堆叠掉落，因「真实不一定好玩」。
Ask：精力在 Prompt / 需求表达，不是敲代码；强调不是一句 Prompt 出成品。
Plan：第 1 周先跑通核心玩法；第 2 周再打磨 PS1 风格、联机与工具链。
Debug：Sandbox 调驾驶手感；长会话修 Bug；地图整城生成失败后改思路。
Agent：2～3 路 Claude 并行；让 AI 做地图/道路/过场编辑器，再用工具继续开发。

原文：36氪报道 · Reddit · 作者 Medium · 试玩：游戏页

🛠️ 同一项目的「造工具」杠杆 Agent 专项

整城地图一次生成失败 → 不硬刚，改为让 Claude 做游戏内编辑器（地形、刷材质、摆模型、自动铺路、过场时间轴）。作者认为最大提效不是那 2.7 万行业务代码，而是这套工具链。对照：Agent 页的「放大器」= 让 AI 提高你下一轮的产能，而不只是多吐几行代码。

🎮 Vibe Jam 赛制本身 Ideate+Plan

Pieter Levels 主办的年度比赛：强制「代码绝大部分由 AI 生成」。这等于赛前就把 Ideate/Plan 的约束写死——你必须用意图驱动，而不是手写堆量。适合用来理解：外部约束也能当创想边界（做什么、不做什么、用什么工具栈）。赛站：vibej.am/2026

📦 资产墙：代码 Agent ≠ 全栈造梦 Plan 专项

业界实测常见结论：Claude Code 等擅长玩法/脚手架，但贴图、Sprite、3D、音频仍要图像/3D/音频模型或外包管线。Capybara 案例也是「代码一路 + 多模态资产多路」。Plan 时就要拆：哪些交给 Coding Agent，哪些交给资产工具——否则会在 Ask 里空耗。
