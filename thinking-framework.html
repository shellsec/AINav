链路还要补上的实战原则
1. 人负责判断，AI 负责执行 — 「100% AI 写代码」说的是 Execute；选题、好玩与否、砍范围、何时停，必须人拍板（Ideate / Debug）。
2. 好玩 > 物理真实 — Ideate/Debug 时可以故意调「不真实」的参数（如食物堆更容易倒），体验优先于仿真。
3. 做不出产物，就先造工具 — Agent 放大器：AI 画不了整张地图 → 让它做地图编辑器 → 你用工具搭。这是 Vibe Coding 最高杠杆之一。
4. 多会话 + 上下文策略 — 新功能用干净会话；修 Bug 用长上下文会话；2～3 路并行各管模块，减少互相踩文件。
5. 把约束写进 Rules / CLAUDE.md — 赛制、编码约定、禁止事项提前固化，减少每次 Ask 重复解释（Capybara 把 VibeJam 规则丢进 /rules）。
6. 模块化提问，别一次「做亚马逊」 — Plan/Ask 拆成可验收小目标；大上下文会「忘」——独立产品里也常见「过长单会话质量掉崖」。
